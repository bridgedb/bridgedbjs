innerenrich(input, options){Stream.<object>}

entity-reference-service.js, line 166

Enrich entity reference. Default is to enrich as much as possible as much as possible with data from BridgeDb, with the exception of not getting xrefs, but the data included for enrichment can be controlled by setting options.include.

Name Type Description
input object | string | stream

Entity reference(s). Each one must have an identifier (e.g. ENSG00000160791) and means for identifying the database.

Acceptable entityReference input arguments:

 1. BridgeDb xrefs URL or identifiers.org IRI as string
 2. Object with at least one of these properties:
   a. { '@id': identifiers.org IRI }
   b. { bridgedbXrefsUrl: BridgeDb xrefs URL }
   c. { xrefs: [
       BridgeDb xrefs URL
       ...
     ]
   }
 3. Object with both of these properties:
   {
     db: database name as specified in datasources.txt
     identifier: entity reference identifier, such as ChEBI:1234
   }

Example of BridgeDb xrefs URL: http://webservice.bridgedb.org/Human/xrefs/L/1234 Example of identifiers.org IRI: http://identifiers.org/ncbigene/1234
