innerenrich(entityReference, options){Stream.<object>}

entity-reference-service.js, line 142

Enrich entity reference. Default is to enrich as much as possible as much as possible with data from BridgeDB, with the exception of not getting xrefs, but the sources of enrichment data can be controlled by setting options.sources.

Name Type Description
entityReference object | string

Entity reference must have an identifier (e.g. ENSG00000160791) and means for identifying the database Acceptable entityReference input arguments:

 1. BridgeDB xrefs URL or identifiers.org IRI as string
 2. Object with at least one of these properties:
   a. { '@id': identifiers.org IRI }
   b. { bridgedbUrl: BridgeDB xrefs URL }
   c. { bridgedbUri: BridgeDB xrefs URL }
   d. { xrefs: [
       BridgeDB xrefs URL
       ...
     ]
   }
 3. Object with both of these properties:
   {
     db: database name as specified in datasources.txt
     identifier: entity reference identifier, such as ChEBI:1234
   }

Example of BridgeDB xrefs URL: http://webservice.bridgedb.org/Human/xrefs/L/1234 Example of identifiers.org IRI: http://identifiers.org/ncbigene/1234
options object optional
Name Type Description
sources Array.<string> optional

Limit enrichment to selected sources of data. Sources currently include "Organism" and "DatabaseMetadata" (metadata about biological databases from datasources.txt). Default is ['Organism', 'DatabaseMetadata']. Entity reference will always be turned into an object, if it is not already an object, and the data provided will be parsed and normalized, even if sources is specified as an empty array.
Returns:

entityReference['@context'] string JSON-LD context. You can ignore this if you don't care about semantics.
entityReference['@id'] string JSON-LD IRI. You can also ignore this.
entityReference.displayName string
entityReference.standardName string
