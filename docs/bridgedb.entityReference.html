innerenrich(input, options){Stream.<object>}

entity-reference.js, line 166

Enrich entity reference. Default is to enrich as much as possible as much as possible with data from BridgeDb, with the exception of not getting xrefs, but the data included for enrichment can be controlled by setting options.include.

Name Type Description
input object | string | stream

Entity reference(s). Each one must have an identifier (e.g. ENSG00000160791) and means for identifying the database.

Acceptable entityReference input arguments:

 1. BridgeDb xrefs URL or identifiers.org IRI as string
 2. Object with at least one of these properties:
   a. { '@id': identifiers.org IRI }
   b. { bridgeDbXrefsUrl: BridgeDb xrefs URL }
   c. { xref: [
       BridgeDb xrefs URL
       ...
     ]
   }
 3. Object with both of these properties:
   {
     db: database name as specified in datasources.txt
     identifier: entity reference identifier, such as ChEBI:1234
   }

Example of BridgeDb xrefs URL: http://webservice.bridgedb.org/Human/xrefs/L/1234 Example of identifiers.org IRI: http://identifiers.org/ncbigene/1234
options object optional
Name Type Default Description
organism boolean true optional

Enrich with organism name.
context boolean true optional

Enrich with JSON-LD @context.
dataSource boolean true optional

Enrich with database metadata (metadata about biological databases from datasources.txt).
bridgeDbXrefsUrl boolean true optional

Enrich with URL for BridgeDb webservices to enable getting xrefs for this entity reference.

Get potential matches for a desired entity reference by free text search for matching symbols or identifiers. See also Java documentation.

Name Type Description
args object
Name Type Description
attribute string

Attribute value to be used as search term
organism string

Organism name. Currently only accepting Latin and English.
type string optional

Entity reference type, such as Protein, Dna, SmallMolecule, etc. Not currently being used, but we might use it in the future to help narrow down the search results.
db string optional

Desired database name, such as Ensembl or Uniprot

Example
myBridgeDbInstance.entityReference.freeSearch({
  attribute: 'Nfkb1',
  organism: 'Mouse'
}).each(function(searchResult) {
  console.log('Result for Nfkb1');

});
